# 嘘つきトーク 🍻

トークの中に、嘘つきが1人。

お酒の場を想定した「お題トーク+正体隠匿」のパーティーゲームです。
全員が同じお題で実体験を語りますが、1人だけ「全部作り話(嘘つき)」。
誰が嘘つきかを当てる、**スマホ1台を回して遊ぶ**ワンターン完結ゲームです。

- 対象人数: 3〜10人
- 1ゲーム = 1ラウンドで完結
- サーバー不要・完全クライアントサイド・オフライン動作

## 遊び方

1. 全員に同じ**お題**が配られ、実体験を話します。ただし1人だけ「全部作り話」の**嘘つき**です。
2. 嘘つきには**本人だけに見える秘密の縛り**が1つ課されます。「7」「まさか」のような具体値が2つ組み合わさった難しい縛り(例:数字の「7」と言葉の「まさか」を両方入れる)で、作り話に無理やり盛り込むと不自然さ=ボロが出ます。正直者に縛りはなく、実体験をそのまま話します。
3. 順番に1人ずつトーク(制限時間あり)→ 質問タイム。市民は「秘密の縛りがある」ことだけ知らされ、無理に盛り込まれた言葉を手がかりに嘘つきを探します。
4. 「嘘つきだと思う人」に一斉投票。
5. 勝敗判定:
   - **最多票が嘘つき** → 市民チームの勝ち
   - **最多票が嘘つき以外、または同票** → 嘘つきの逃げ切り勝ち

## 技術構成

- React + Vite(完全静的・外部API通信なし)
- 状態管理は `useReducer` / `useState` のみ(localStorage 不使用・全てメモリ内)
- スマホ縦画面前提のレスポンシブUI(375px幅基準・ダークテーマ)

### ディレクトリ

```
src/
  App.jsx              フェーズ管理・全体配線
  game.js              状態遷移(reducer)・抽選・勝敗判定
  data.js              お題(46個)・縛りルール(8種、嘘つき専用の複合縛り)
  useCountdown.js      カウントダウンタイマー
  components/          各画面(Title / Setup / Reveal / Talk / Question / Vote / Result)
```

## ローカル実行

```bash
npm run preview  # ビルド結果のプレビュー
```

## Vercel へのデプロイ

完全な静的サイトなので、サーバーレス関数も環境変数も不要です。

1. このリポジトリを GitHub に push する
2. [Vercel](https://vercel.com/) で「New Project」→ 当リポジトリを Import
3. Vite プロジェクトとして自動検出されます(手動設定する場合は以下)
   - **Framework Preset**: Vite
   - **Build Command**: `vite build`
   - **Output Directory**: `dist`
4. Deploy を実行

以降は `main` ブランチへの push で自動デプロイされます。
デプロイ完了後、URL を上記「デプロイURL」に記載し、QRコードを生成して共有してください。

## 注意

- リロードするとゲームの進行状態は消えます(全てメモリ内管理のため)。
